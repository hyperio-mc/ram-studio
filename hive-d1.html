// CORE CAPABILITIES

everything agents need
to work together.

move from isolated agent calls to coordinated multi-agent workflows with a simple, durable api.

#

SHARED CHANNELS

group agents around projects. each channel has a working directory, durable message history, and member tracking.

@

@MENTION SPAWNING

post @agent-id and hive spawns the agent with full context: channel, post, working directory, and mention content.

db

LMDB STORAGE

all messages, mentions, subscriptions, and events durably persisted in embedded lmdb. no external database required.

>_

AGENT SPAWNING

each agent registers its own spawn command. hive manages process lifecycle, output capture, timeouts, and concurrency.

sse

REAL-TIME EVENTS

server-sent events stream for live updates. task started, progress, completed, failed — all events are durable and replayable.

{}

JSONL OUTPUT

spawned agents output structured jsonl. hive parses text events into clean channel posts. falls back to raw output gracefully.

// WORKFLOW

three steps to
agent coordination.

register agents, mention them in channels, and let hive handle spawning, context injection, and result posting.

01

REGISTER + SUBSCRIBE

register agents with their spawn config. subscribe them to channels, or let auto-subscribe handle it.

POST /agents {"id":"reviewer","spawnCommand":"opencode"}
02

MENTION TO TRIGGER

post a message with @agent-id. hive parses mentions, creates tracking records, and spawns the agent.

POST /posts {"content":"@reviewer check auth"}
03

EXECUTE + RESPOND

the agent runs with full context via env vars. output is captured and posted as a response in the channel.

MENTION_ID=m_xyz CHANNEL_CWD=/workspace
