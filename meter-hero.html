ORIGINAL DESIGN PROMPT

"Design METER — a developer-first AI token cost intelligence platform for engineering teams. Inspired by Polar.sh's billing pipeline aesthetic (land-book.com) and Obsidian OS (darkmodedesign.com). Terminal event streams, usage metering, cost anomaly detection, multi-model comparison. Zinc dark + electric blue palette. First time building terminal-as-UI aesthetic."

PRODUCT OVERVIEW

METER is an AI-native cost intelligence platform for engineering teams spending on LLM APIs. As AI infrastructure costs become a critical line item, engineering managers and FinOps teams need real-time visibility into token consumption, cost anomalies, and budget utilization — with the same precision and density that developers expect from their observability tools.

INSPIRATION

Polar.sh (spotted on land-book.com) — "Turn Usage Into Revenue" — developer-first billing with terminal code blocks as primary hero UI, numbered pipeline sections (— 01, — 02, — 03), live event feeds showing real-time API activity, near-black + electric blue, and a billing pipeline visualization (CHECKOUT → USAGE EVENTS → METERING → INVOICING). Obsidian OS (darkmodedesign.com) — dark finance SaaS for financial advisers with structured AI-powered feature panels, sidebar nav, clean metric tiles. TRIONN (darkmodedesign.com) — bold counter-stat numerics (50+ awards, 900+ projects) on dark backgrounds.

TARGET USERS

  • Engineering Managers — budget owners tracking AI API spend across teams and features
  • Platform Engineers — responsible for LLM infrastructure, routing, and cost optimization
  • FinOps / Finance — monthly AI spend reconciliation and budget forecasting
  • Technical Founders — watching burn rate on AI features in early-stage products

CORE FEATURES

  • Cost Overview — real-time total spend vs. budget with per-model breakdown and period comparison
  • Usage Breakdown — daily token volume charts, cost by feature/team/model with drill-down
  • Alert Center — rule-based budget alerts, cost anomaly detection (spike analysis), spend cap enforcement
  • Live Event Stream — terminal-style real-time inference feed with cost-per-call visibility
  • Cost Explorer — horizontal bar charts by dimension (model/feature/team), model price comparison table
  • Savings Engine — automatic recommendations to route workloads to cheaper equivalent models

DESIGN LANGUAGE

  • Terminal-native aesthetic — monospace event streams as first-class visual chrome, not decoration
  • Zinc dark palette — near-black #09090b base, layered zinc surfaces (zinc-900, zinc-800), zinc-700 borders
  • Semantic color system — blue = normal inference, emerald = savings/hit, amber = warning, red = critical/anomaly
  • Pipeline visualization — Polar-inspired INGEST → METER → BUDGET → ALERT flow as persistent mental model
  • High data density — stat tiles, stacked bar charts, horizontal cost bars — no decorative illustrations

SCREEN ARCHITECTURE

  • M1 — Cost Overview · Home screen with total spend, budget bar, model tiles, live event feed
  • M2 — Usage Breakdown · Daily token bar chart + feature cost list with color-coded top bars
  • M3 — Alert Center · Severity-coded alert cards with inline action buttons (Critical/Warning/Info)
  • M4 — Live Event Stream · Terminal window full-screen with INF/ERR/ALT/CAC/EMB event types
  • D1 — Main Dashboard · Bento grid: 4 stat tiles + model chart + pipeline + alert + live stream strip
  • D2 — Cost Explorer · Horizontal cost-by-feature chart + model comparison table + savings card

WHAT'S NEW IN THIS DESIGN

First RAM design to use terminal/monospace as primary visual language — not just accent. The Live Event Stream screen treats a scrolling log as the hero UI. First to visualize a billing pipeline (INGEST → METER → BUDGET → ALERT) as a permanent orientating diagram. First time mixing engineering observability patterns (event types: INF/ERR/ALT/CAC/EMB) with consumer-grade mobile UX.
